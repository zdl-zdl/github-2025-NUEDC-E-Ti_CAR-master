# NUEDC Ti_CAR 接线与调试说明

> 适用板卡：MSPM0G3507（天猛星/庐山派配套核心板） + **TB6612 双路电机驱动模块**（图示型号：TB6612FNG 双路板） + MG310 编码器（13 线）+ 8 路灰度
> 波特率：115200（UART0，PA0=TX / PA1=RX）

---

## 0. 上电行为（1m 正方形循迹·按键发车）

当前 `App/utils.c` 默认**上电等待按键发车**，按 PB9 键后开始循迹 1m 正方形，跑 1 圈（4 个拐角）后自动停车：

```c
uint8_t lap_num          = 1;   // 1圈=4个拐角，跑完自动停车
uint8_t task_num         = 1;   // FirstTask（循迹 + 90°转弯）
uint8_t total_start_flag = 0;   // 0=等按键发车(按PB9)；1=上电直接发车
uint8_t wait_flag        = 1;   // 按键后直接走，跳过倒计时
uint8_t debug_no_turn    = 0;   // 0=正常拐角转弯（正方形必须启用！）
uint8_t debug_motor_stop = 0;   // 0=电机正常工作
```

- **上电后车不动**，串口每 ~500ms 打印一行（等待按键）：
  `GR:0 0 1 1 1 0 0 0 | EL:0 ER:0 | VL:0.0 VR:0.0 | bs:0 | dL:0 dR:0`
- **按 PB9 键** → 开始循迹，串口打印变为 `bs:280`
  - `GR` = 8 路灰度（0=白/1=黑），`EL/ER` = 编码器累计脉冲，`VL/VR` = 轮速(m/s)，`bs` = 基础速度，`dL/dR` = 距离(mm)
- 跑完 4 个拐角后自动停车（`lap_num=1`）。
- 想上电直接发车：把 `total_start_flag` 改回 `1`。

| TB6612 板子 **3V3 排针**（500mA） | （空着/备用） | 板子输出的 3.3V，本项目由 MSPM0 自供 3.3V 即可 |

> ⚠️ **编码器走独立线**：双路板上的 E1A/E1B/E2A/E2B 是板上拉 5V 的，MSPM0 GPIO 不 5V tolerant。**强烈建议编码器不走板载通道，而是用 4 根独立线**接到 MSPM0，详见 §3。

### 2.2 STBY 引脚处理（新板必看）

这块 TB6612 双路板比原四驱板多了一个 **STBY（Standby，待机）** 控制端：

- **STBY = 高**：TB6612 正常工作
- **STBY = 低（或悬空）**：TB6612 进入待机，**所有电机都不会转**！

当前代码没用到 STBY 引脚，所以必须把它**拉高**使能。两种方案：

**方案 A（推荐，简单）：直接短接到 3.3V**
- 用杜邦线把 STBY 接到板子 **3V3 排针**（板子自输出的 3.3V）或 MSPM0 3.3V
- 缺点：拔掉 STBY 这根线电机就停，不能软件控制

**方案 B（调试友好）：接一个 MSPM0 空闲 GPIO**
- 比如 PA7 / PA8 / PA24 / PA25（找没被占用的）
- 上电时 `DL_GPIO_setPins(...)` 拉高，调试时可 `Motor_Stop_all()` 软断电
- 改 `empty.syscfg` 新增一个 GPIO 实例，名字 `STBY`，设输出高

> 抽空建议用方案 B，但**当前没接 STBY、电机不转**就是这个引脚没拉高引起的 99% 问题。

### 2.3 电机接口（白色 6P 插座）

每个电机接口是 6P 插座，引脚顺序（从图示板上读出）：

```
电机A接口（6P）：  AO1  5V  E1B  E1A  GND  AO2
电机B接口（6P）：  BO1  5V  E2B  E2A  GND  BO2
```

| 板子丝印 | 接到 | 说明 |
|---|---|---|
| **AO1** | 左电机 **红线** | 电机A电源 + |
| **AO2** | 左电机 **黑线** | 电机A电源 - |
| **BO1** | 右电机 **红线** | 电机B电源 + |
| **BO2** | 右电机 **黑线** | 电机B电源 - |
| **5V（电机口）** | 空着 | 仅给编码器供电，本项目用独立线 |
| **E1A/E1B/E2A/E2B（电机口）** | 空着 | 编码器走独立线，不走板载 |
| **GND（电机口）** | 空着 | 共地已经在单片机接口 GND 处接了 |

> 接电机线时**只接 AO1/AO2 和 BO1/BO2 这 4 根**，中间 4 针（5V/E1A/E1B 等）**留空**，避免误碰 5V 进 MSPM0。

## 3. MG310 编码器 ⟷ MSPM0（走独立线，强烈推荐）

> ⚠️ **为什么不用板载 E1A/E1B/E2A/E2B 通道**：
> - 板子给编码器上拉到 5V（电机口 5V → 编码器 → 板载上拉 → 单片机接口端 E1A）
> - MSPM0G3507 GPIO 不是 5V tolerant，长期接 5V 有损坏风险
> - 用独立线 + MSPM0 内部上拉（已配），3.3V 干净稳定

|---|---|---|
| PB4 | 左轮 编码A | 绿/白 |
| PB5 | 左轮 编码B | 白/绿 |
| PB23 | 右轮 编码A | 绿/白 |
| PB12 | 右轮 编码B | 白/绿 |
| 3.3V（MSPM0 自供） | 红 | 供电（**不要接板子 5V**） |
| GND | 黑 | 与板子 GND 共地 |

> 四路编码器 GPIO 均已配内部上拉。方向反了就把 `Hardware/encoder.c` 里对应 `++ / --` 对调。

## 7. 推荐接线顺序

1. 接 **所有 GND + 电源**：电池 → DC 座、MSPM0 3.3V → 编码器/灰度、TB6612 GND → MSPM0 GND。**万用表测 DC 座对地无短路**。
2. 接 **TB6612 单片机接口**：PWMA/PWMB/AIN1/AIN2/BIN1/BIN2/STBY → MSPM0。**STBY 这步不要忘**（短接 3.3V 即可）。
3. 接 **TB6612 电机口**：AO1/AO2/BO1/BO2 → 电机 4 根线，**中间 4 针留空**。
4. 接 **编码器独立线**：4 根信号线 → PB4/PB5/PB23/PB12，VCC→3.3V、GND→共地。
5. 接 **8 路灰度**：VCC/GND/DO → 3.3V / 共地 / PA27 PB25 PB20 PA14 PA16 PA15 PA10 PA11。
6. 接 **USB-TTL**：PA0/PA1/GND，先不接 K230。
7. **拨动开关拨 OFF**，插电池。
8. 上电看串口是否有打印 → 正常后**拨动开关拨 ON**，电机会得电。

## 8. 抬车测编码器（debug_motor_stop=1）

1. `App/utils.c` 里 `debug_motor_stop` 改成 **1**，编译下载。
2. 上电，**车抬离地、电机不转**，串口看 `EL/ER`。
3. 手正转左轮 → `EL` 增大；反转 → 减小。右轮看 `ER`。
   - 不动：查 PB4/PB5/PB23/PB12 接线、供电、上拉。
   - 方向反：对调 `encoder.c` 中 `++/--`。
4. 用螺丝刀/黑胶带分别挡 8 路灰度，对应 `GR` 位应从 0 变 1。
5. 测完 `debug_motor_stop` 改回 0，`debug_no_turn` 设 0（启用转弯跑正方形）。

## 9. 1m 正方形循迹逻辑

### 9.1 运行流程

```
上电 → System_Init() → 主循环 TaskSelect() → FirstTask()
  │
  ├─ workstep 0: 初始化 PID(145/0/65)、basespeed=280
  │
  └─ workstep 1: 循环执行
       │
       ├─ [循迹阶段] TRACK_PID：灰度传感器跟线直行
       │   └─ Track_DetectCorner()：≥4路灰度压黑线 → corner_detected=1
       │
       ├─ [转弯阶段] TURN_90_PID：左轮停、右轮转，定点 90°
       │   └─ fabs(carR_dis) >= TURN_DISTANCE → 转弯完成，恢复循迹
       │
       └─ 第4个拐角后：lap_count(1) >= lap_num(1) → 最终转弯 → Motor_Stop() 停车
```

### 9.2 拐角检测条件
- **≥4 路**灰度传感器同时检测到黑线
- 连续 **3 次**主循环扫描满足上述条件
- 消抖计时 ≥100 次主循环迭代
- 满足后：`corner_detected=1`，`corner_cnt++`，蜂鸣器+LED 反馈
- 每 **4 个拐角** = 1 圈（`lap_count++`）

### 9.3 90° 转弯机制

**当前：编码器开环定点转弯**
- 左轮停（PWM=0）、右轮前进（bias=190 + 速度PID），车绕左轮 pivot 转
- 右轮前进距离由编码器测量：`carR_dis = WHEEL_DIST × 1000 × (-encoder_right_count)`
- 当 `IsTurnComplete()` 返回 true 时转弯完成
- 转弯参数：`basespeed=20`（速度目标）、`bias=190`（PWM 偏置）

**开环转弯的问题（过度转弯/转歪的原因）**：
1. **打滑**：定点转弯左轮拖地，编码器计了数但车没转那么多 → 实际转角 < 预期
2. **速度不受控**：速度环目标 20 m/s 不可达，实际靠 bias=190 固定 PWM，电池电压变化影响转速
3. **每次转弯不一致**：无角度反馈，重复性差

### 9.4 转弯改进方案

**方案 A：差速转弯（减少打滑，无需陀螺仪）**
- 左轮反转 + 右轮正转，原地转弯
- 优点：两轮都转不拖地，打滑少，重复性好
- 注意：`TURN_DISTANCE` 需减半（差速转弯 = 轮距×π/4，定点转弯 = 轮距×π/2）
- 代码见 `Control/speed.c` 的 `turn_90_control` 注释

**方案 B：陀螺仪闭环（最精确，后续接入）**
- 用陀螺仪 yaw 角度作为转弯反馈
- 代码已预留，结构如下：
  - `App/task.h`：`TurnMode_e` 枚举（`TURN_ENCODER` / `TURN_GYRO`）
  - `App/task.c`：`IsTurnComplete()` 函数，陀螺仪模式用 yaw 误差 < 3° 判断
  - `App/task.c`：`Turn_Start()` 函数，转弯开始时记录 `turn_target_yaw = gyro_yaw + 90`
- **接入步骤**：
  1. 在定时器中断里读取陀螺仪 yaw，更新 `gyro_yaw` 变量
  2. 把 `turn_mode` 改成 `TURN_GYRO`
  3. （可选）`turn_90_control` 里用 yaw 误差 PD 控制转弯速度，替代固定 bias

### 9.7 TURN_DISTANCE 调试指南

`TURN_DISTANCE` 是 90° 定点转弯时右轮前进的距离（单位 mm），直接影响转弯角度精度。

**理论计算**：
```
TURN_DISTANCE = 轮距(mm) × π / 2
```
- 当前值 320mm → 对应轮距 ≈ 204mm（20.4cm）
- **用尺子量你的车左右轮中心间距**，代入公式得到理论值

**实测调参步骤**：
1. 抬车或放空旷平地，标记车头朝向
2. 上电运行，等车完成一次 90° 转弯后停车
3. 观察转弯角度：
   - **转不够（<90°）** → `TURN_DISTANCE` **调大**
   - **转过冲（>90°）** → `TURN_DISTANCE` **调小**
4. 串口 `dR` 值显示右轮实际转弯距离，对比 `TURN_DISTANCE` 确认
5. 反复微调直到转正 90°

## 10. 常见问题速查

| 现象 | 原因 | 解决 |
|---|---|---|
| **电机完全不转** | 99% 是 STBY 没拉高 | 用杜邦线把 STBY 短接到 3V3 排针 |
| **电机一个转一个不转** | 检查 6 路信号是否都接对：AIN1/AIN2/PWMA 为一组，BIN1/BIN2/PWMB 为一组 | 对照 §2.1 表格 |
| **串口没数据** | `fputc` 重定向不在工程里 | 确认 `Bsp/uart.c` 在 Keil 工程中（含 fputc 代码） |
| **串口发数据没回显** | UART0 RX 中断抢先读走数据 | 测试时注释掉 `UartDeviceInit()`，用轮询 |
| **转弯过度/转歪** | 开环转弯打滑 + 速度不受控 | 见 §9.4 改差速转弯或接入陀螺仪 |
| **上电不走** | `total_start_flag=0` 等按键 | 按 PB9 键发车，或改回 `1` |
| **转弯角度不对** | `TURN_DISTANCE` 未校准 | 见 §9.7 调试指南，量轮距算理论值再微调 |
| **不转弯（直冲）** | `debug_no_turn=1` | 改为 0（`App/utils.c`） |
| **跑完不停车** | `lap_num` 太大 | 改为 1（`App/utils.c`） |
| **PWM 输出 0** | 串口打印 `VL:0.0 VR:0.0` | PID 算出来就是 0，或 `debug_motor_stop=1` 忘了改回 0 |
| **方向反** | AIN1/AIN2 或 BIN1/BIN2 接反 | 对调 `motor.c` 中 `AIN1_OUT(0)/AIN1_OUT(1)` |
| **编码器读数不动** | 编码器独立线没接好 | 检查 PB4/5/23/12、3.3V、GND 6 根线 |
| **编码器读数乱跳** | 走板载 E1A/E1B/E2A/E2B（5V 上拉） | 改回独立线方案（见 §3 警告） |
| **烧了第一颗芯片** | 5V 进 MSPM0 任意引脚 | 检查所有 5V 排针都空着 |
| **TB6612 烫手** | 长时间堵转 | 降低 `basespeed` 或加软启动 |
